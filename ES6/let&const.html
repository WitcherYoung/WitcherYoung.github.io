# let

# 1. let实现了块级作用域

先来简单说说块级作用域

在ES5, JavaScript只存在函数作用域, 每次函数执行的时候都会生成一个AO对象

函数作用域内中 定义的 变量和函数 会被先解析成AO对象的属性, 然后形参与实参相统一, 函数体与函数名相统一, 再执行函数中的语句

因此, 不管在哪里用var声明变量, 函数执行预编译都会将var声明变量提到函数顶部, 所以变量先使用后定义是不会报错的, (如果是声明时赋值, 那么就相当于在顶部声明了一个值为undefined的变量, 引擎读到这句时再赋值)

而ES6 使用let声明的变量不存在变量提升, 作用域则是包含该变量的一对"{}"内并且let声明语句以下部分,实现了块级作用域的概念

# 2. 使用let声明变量不存在变量提升, 所以会存在暂时性死区的问题, 在声明结束后才能操作变量,否则会出现ReferenceErr报错

下例中 会出现ReferenceErr报错

var test  = 111, tmp = 123;
if (true) {
  test = 'aaa';
  tmp = 'abc'; // ReferenceError
  /*
  *  只要块级作用域内存在let命令，它所声明的变量就“绑定”（binding）这个区域，
  *  不再受外部的影响。
  */ 
  let tmp;
}
function bar(x = y, y = 2) {
    return [x, y];
}
bar(); // 报错

在函数参数指定默认值时也一定要遵循先声明再赋值的原则, 下列未声明就赋值的例子也是暂时性死区的一种 互换位置便可解决此问题

function bar(x = y, y = 2) {
  return [x, y];
}

bar(); // 报错

# 3. 使用 let 时不允许在相同作用域内重复声明变量

下例中 会出现SyntaxError报错

function fn() {
    var a = 10;
    let a = 10;
    console.log(a);
}
fn();

# 4. 在同一作用域内, 使用let声明的变量, 是不允许被重复声明的, 也不允许在函数内重新声明参数

下例中let声明的a变量是不允许被重复声明的

// 报错
let a = 10;
var a = 1;
// ------
// 报错
let a = 10;
var a = 1;
// ------
// 如果使用var声明, 是没有问题的
var a = 10;
var a = 1;
// ------
// 在函数体中, 使用let声明的变量与参数重名, 也会报错, 属于重复声明变量
function func(arg) {
  let arg;
}
func()
// ------
// 不报错
function func(arg) {
  {
    // 此时声明的变量在func函数内部的一个块级作用域内, 不会报错
    let arg;
  }
}
func()
// ------

tips:

let实现块级作用域, 可以避免函数作用域内变量泄露到全局作用域, 避免访问全局变量时被用var定义的变量覆盖从而导致访问到的变量值为undefined

ES6 允许块级作用域的任意嵌套

{{{{
 {let insane = 'Hello World'}
 console.log(insane); // 报错
}}}};

内层作用域可以定义外层作用域的同名变量

{{{{
 let insane = 'Hello World';
 {let insane = 'Hello World'}
}}}};

块级作用域的存在可以替代立即执行函数  

值得注意的是

ES6 引入了块级作用域，明确允许在块级作用域之中声明函数。ES6 规定，块级作用域之中，函数声明语句的行为类似于let，在块级作用域之外不可引用。但是 如果改变了块级作用域内声明的函数的处理规则，显然会对老代码产生很大影响。为了减轻因此产生的不兼容问题

ES6 在附录 B里面规定，浏览器的实现可以不遵守上面的规定，有自己的行为方式

  • 允许在块级作用域内声明函数。
  • 函数声明类似于var，即会提升到全局作用域或函数作用域的头部。
  • 同时，函数声明还会提升到所在的块级作用域的头部

注意 上面三条规则只对 ES6 的浏览器实现有效，其他环境的实现不用遵守，还是将块级作用域的函数声明当作let处理。

根据这三条规则，在浏览器的 ES6 环境中，块级作用域内声明的函数，行为类似于var声明的变量。

# const

# 1. const用来声明常量, 声明的值是不可改变的, const一旦声明变量就必须初始化(赋值), 如果只声明不赋值, 就会报错

这里有一点需要注意的是

  • 当声明的变量为基础数据类型, 那么该变量的值是不可被改变的

  • 当声明的变量为引用类型, 那么该变量引用的索引是不能被改变的, 也就是说永远只能指向定义时的对象(或数组), 但是 该变量的属性值是可以被更改的( 对象的属性, 数组长度和元素)

所以将 对象 声明为常量是需要用 Object.freeze 方法将对象冻结

# 2. const的作用域与let命令相同：只在声明所在的块级作用域内有效

# 3. const命令声明的常量也是不提升，同样存在暂时性死区，只能在声明的位置后面使用

# 4. const声明的常量，也与let一样不可重复声明

# let const与var的区别

var命令和function命令声明的全局变量，依旧是顶层对象的属性；另一方面规定，let命令、const命令、class命令声明的全局变量，不属于顶层对象的属性。也就是说，从 ES6 开始，全局变量将逐步与顶层对象的属性脱钩

var a = 1;
// 如果在 Node 的 REPL 环境，可以写成 global.a
// 或者采用通用方法，写成 this.a
window.a // 1

let b = 1;
window.b // undefined
// 上面代码中，全局变量a由var命令声明，所以它是顶层对象的属性；全局变量b由let命令声明，所以它不是顶层对象的属性，返回undefined
