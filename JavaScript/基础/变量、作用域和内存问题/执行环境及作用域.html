# 执行环境( 执行上下文 )及作用域

执行环境( execution context )是JavaScript中最为重要的一个概念

每个 执行上下文 都有一个与之关联的变量对象( variable object ), 执行上下文 中定义的所有变量和函数都保存在这个对象中, 当 执行上下文 中所有代码执行完毕之后, 该环境就被销毁了( 其中的所有变量及函数定义也被销毁 )

我们编写的代码是无法访问这个对象的, 解析器在处理数据时会在后台使用它, 这对我们了解 JavaScript 是如何运行的, 是极其有帮助的

全局执行上下文是最外围的一个执行环境, 根据ECMAScript实现所在的宿主环境的不同, 全局执行上下文也是不一样的

在浏览器中, 全局执行上下文被认为是 window 对象, 也就是说, 所有全局变量和函数都是作为window对象的属性和方法创建的

# 每个函数都有自己的执行上下文

当 执行流 进入一个函数时, 函数的执行上下文就会被推入到 执行上下文栈中, 在函数执行完毕后, 执行上下文栈 将 执行完毕的函数执行上下文弹出, 代码继续向下执行

# 思考

# 作用域链可以理解为执行上下文栈?

首先我们要明白

# 执行上下文栈 是显示 函数的变量对象的进栈顺序

也就是说 执行上下文栈 是表示 执行流 进入函数执行的顺序

# 作用域链是 在一个函数执行时 访问一个本作用域内不存在的变量 向上一级作用域寻找的依据

作用域链则是在函数定义时就已经决定了, 函数执行位置是不会影响函数作用域及作用域链的

结合下图具体说明

function out1(count1) {
    function out1Inner1() {
        var inner1 = 1;
        console.log(`out1Inner1: ${count1}`);        // 访问到out1函数的参数count1
    }
    function out1Inner2(count1) {
        var inner2 = 1;
        console.log(`out1Inner2: ${count1}, ${inner2}`);
        /** 
         * out1Inner2存在参数count1, 此处访问到out1Inner2函数的参数count1, 
         * 及out1Inner2内部inner2 
         */
        out2();
        /** 
         * 此处执行out2, out2函数体内依然是无法访问到count1, inner2变量的
         * 此处会报错
         */
    }
    out1Inner1(1);
    out1Inner2(2)
}

function out2(count2) {
    function out2Inner1() {
        var inner3 = 1;
        console.log(`out2Inner1: ${count1}, ${inner2}`);
        /** 
         * out2Inner1不存在参数count1, inner2, 
         * 其函数作用域链为 out2Inner1 -> out2 -> global 作用域链中都不存
         * 此处会报错
         */
    }
    out2Inner1();
}

out1("out1");
out2("out2");

以上例子可理解为对象层级, 如下图解

从中可以发现, 一个函数的作用域链只会沿着当前作用域往上寻找 即 out2Inner1 -> out2 -> global

就算是同级作用域 也不能相互访问
