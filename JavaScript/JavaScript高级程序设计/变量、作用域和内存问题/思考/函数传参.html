# 参数传递

ECMAScript 中所有函数的参数都是按值传递的。

但是我对此处的例子来解释引用类型是按值传递的, 仍有些疑问

function setName(obj) {     
  obj.name = "Nicholas";     
  obj = new Object();     
  obj.name = "Greg"; 
} 

var person = new Object(); 
setName(person); 
alert(person.name);    //"Nicholas"

书中对此例给的解释是这样的

一行代码为 obj 重新定义了一个对象，

另一行代码为该对象定义了一个带有不同值的 name 属性。在把 person 传递给 setName()后，其name 属性被设置为"Nicholas"。

然后，又将一个新对象赋给变量 obj，同时将其 name 属性设置为"Greg"。

如果 person 是按引用传递的，那么 person 就会自动被修改为指向其 name 属性值 为"Greg"的新对象。

但是，当接下来再访问 person.name 时，显示的值仍然是"Nicholas"。

这说明 即使在函数内部修改了参数的值，但原始的引用仍然保持未变。

实际上，当在函数内部重写 obj 时，这 个变量引用的就是一个局部对象了。

而这个局部对象会在函数执行完毕后立即被销毁。

但是我认为书中对此例的解释有些欠妥, 以下是我的看法

person 使用Object构造函数赋值时 为一个空对象

把 person 传给setName()后, obj参数保存的是 person 对象的地址

( 因为 此处 obj.name = "Nicholas", 在最后输出 person.name 也是"Nicholas", 所以我认为这是按引用传递的 )

接着到了最关键的一条语句 obj = new Object(); ( 我认为和作者最大的分歧是这里 )

这条语句 用 Object 构造函数 重新为 obj 赋值, 在堆内存中新构造了一个空对象, 然后 obj 参数保存的 指向 person 的地址, 被这个新对象的地址给替换掉了

这样就导致了 obj指向的地址 与 person指向的地址 不是同一个

所以最后 obj.name = "Greg"; 修改的 name 属性值是函数内部新构造对象上的

这时 obj 与 person 已经不存在关联了, 此处的赋值语句自然也就影响不了person 了

为了验证自己的想法, 我对上述例子加以改造并对每条语句新增控制台输出语句

function setName(obj) { 
  console.log(`obj.name"; ${obj.name}`);    
  console.log(`before obj.name = "Nicholas"; ${person.name}`);    
  console.log(`--------------------------------------------`);
  obj.name = "Nicholas"; 
  console.log(`after  obj.name = "Nicholas"; ${person.name}`);    
  console.log(`before obj = new Object();    ${person.name}`);    
  console.log(`--------------------------------------------`);
  obj = new Object(); 
  console.log(`after  obj = new Object();    ${person.name}`);    
  console.log(`before obj.name = "Greg";     ${person.name}`);    
  console.log(`--------------------------------------------`);
  obj.name = "Greg"; 
  console.log(`after  obj.name = "Greg";     ${person.name}`);    
  console.log(`--------------------------------------------`);
}

var person = new Object(); 
person.name = "test name";
setName(person); 
console.log(`after  setName(person);       ${person.name}`);

输出结果如下

obj: { name: 'test name' }

before obj.name = "Nicholas"; test name

after obj.name = "Nicholas"; Nicholas

before obj = new Object(); Nicholas

after obj = new Object(); Nicholas

before obj.name = "Greg"; Nicholas

after obj.name = "Greg"; Nicholas

after setName(person); Nicholas

--------------------------------------------------------------- 分割线 ----------------------------------------------------------------

接着我又验证了不在函数传参的情况

var person = new Object(), obj = person;
obj.name = "Nicholas";
console.log(`after obj.name = "Nicholas";`);
console.log(`person: `, person);
console.log(`obj:    `, obj);
console.log("----------------");

obj = new Object();
console.log(`after obj = new Object();`);
console.log(`person: `, person);
console.log(`obj:    `, obj);
console.log("----------------");

obj.name = "Greg";
console.log(`after obj.name = "Greg";`);
console.log(`person: `, person);
console.log(`obj:    `, obj);

after obj.name = "Nicholas";

person: { name: 'Nicholas' }

obj: { name: 'Nicholas' }

after obj = new Object();

person: { name: 'Nicholas' }

obj: {}

after obj.name = "Greg";

person: { name: 'Nicholas' }

obj: { name: 'Greg' }

在不调用函数的情况下, 引用类型赋值表现的结果与书中的例子相一致

因此我认为 函数传参还是引用传参理解比较易懂

new Object() 是改变了变量中保存的对象地址, 将变量保存的对象地址指向了一个Object构造函数产生的一个全新的空对象
